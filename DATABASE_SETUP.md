# 🗄️ إعداد قاعدة البيانات (مرة واحدة فقط)

## 🎯 الهدف
هذا الملف يحل **جميع** المشاكل دفعة واحدة:
- ✅ إصلاح خطأ "يوجد يوم عمل مسجّل بالفعل"
- ✅ إصلاح خطأ "Could not find table archived_clients"
- ✅ تأمين البيانات (كل مستخدم يشوف بياناته فقط)

---

## 📋 خطوة واحدة فقط

### 1. افتح Supabase
- روح على: https://app.supabase.com
- اختار مشروعك (artisan-ledger)

### 2. افتح SQL Editor
- من القائمة الجانبية ← **SQL Editor**

### 3. نفّذ Setup الكامل
- افتح الملف: `supabase/COMPLETE_SETUP.sql`
- انسخ **كل** محتويات الملف
- الصقها في SQL Editor
- اضغط **Run** (أو Ctrl + Enter)

---

## ✅ تأكد من النجاح

بعد تشغيل الـ SQL، يجب أن ترى في النتيجة:

```
✅ SETUP COMPLETE!
tablename          | rls_enabled
-------------------+------------
archived_clients   | true
artisan_days       | true
artisan_payments   | true

=== Constraints ===
artisan_days       | artisan_days_user_date_unique
archived_clients   | archived_clients_user_id_client_name_key
```

إذا شفت هذا، يبقى **تمام** كل حاجة شغالة! 🎉

## ⚠️ ملاحظات مهمة

### تشغيل مرة واحدة فقط
- نفّذ الـ SQL **مرة واحدة** فقط
- إذا شغلته مرتين، مافيش مشكلة (آمن)
- الـ script ذكي ويتعامل مع الحالات المختلفة

### لا يؤثر على البيانات
- ✅ البيانات الموجودة محفوظة 100%
- ✅ لن يتم حذف أي شيء
- ✅ فقط إصلاحات وتحسينات

### إذا واجهت مشكلة
1. تأكد نسخت **كل** الملف (من أول سطر لآخر سطر)
2. تأكد أنك في **المشروع الصحيح**
3. تأكد لديك **صلاحيات admin**
4. شوف الـ **Logs** في Supabase لو في خطأ

## 🔍 ماذا يفعل الـ Setup؟

### الجزء 1: RLS Policies
يفعّل Row Level Security للأمان:
```
- كل مستخدم يشوف بياناته فقط
- لا يمكن الوصول لبيانات مستخدمين آخرين
```

### الجزء 2: Fix Unique Constraint
يصلح مشكلة التاريخ:
```
قبل: UNIQUE (date) ❌
بعد: UNIQUE (user_id, date) ✅
```

### الجزء 3: Archive Table
ينشئ جدول الأرشيف:
```
- جدول جديد: archived_clients
- مع RLS policies
- مع indexes للأداء
```

## 🆘 حل المشاكل الشائعة

### خطأ: "permission denied"
**الحل:** تأكد أنك صاحب المشروع أو لديك صلاحيات admin

### خطأ: "relation already exists"
**الحل:** الجدول موجود بالفعل، وهذا طبيعي. تجاهل الخطأ.

### خطأ: "syntax error"
**الحل:** تأكد أنك نسخت الملف كاملاً بدون تعديل

### لا تزال المشكلة موجودة
**الحل:**
1. أعد تشغيل التطبيق: `npm run dev`
2. امسح cache المتصفح: Ctrl + Shift + R
3. سجل خروج ثم دخول
4. جرب من متصفح آخر

## 📞 للمساعدة

إذا استمرت المشكلة:

1. **تحقق من Supabase Logs:**
   - في Dashboard → Logs
   - ابحث عن أي أخطاء

2. **تحقق من Browser Console:**
   - اضغط F12
   - شوف إذا في أخطاء

3. **تحقق من الاتصال:**
   - تأكد الإنترنت شغال
   - تأكد `.env.local` صحيح

4. **شغل التحقق:**
   ```sql
   -- في Supabase SQL Editor
   SELECT * FROM public.archived_clients LIMIT 1;
   ```
   - إذا ظهرت نتيجة أو "0 rows" = تمام ✅
   - إذا ظهر "table does not exist" = المشكلة لسه موجودة ❌
